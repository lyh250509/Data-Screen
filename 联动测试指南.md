# 🎯 二三维联动测试指南

## ✅ 已实现的视觉联动功能

### 1️⃣ 地图 → 3D场景联动

**功能**：点击地图上的点位，3D场景会自动响应

**效果**：
- ✨ 3D场景相机**自动飞行**到对应位置
- 📍 在3D场景中**显示3D标记**（圆柱+球体）
- 🎬 标记会有**上下浮动**和**旋转动画**
- 💚 人员标记是绿色，车辆标记是紫色，设备标记是蓝色
- ⏱️ 飞行动画时长1.5秒，带缓动效果

**测试步骤**：
```
1. 打开浏览器访问 http://localhost:5174/
2. 等待数据加载（控制台显示"✅ 模拟数据已生成"）
3. 在2D地图上找到彩色的圆点（人员/车辆标记）
4. 点击任意一个点
5. 观察3D场景的变化：
   - 相机开始平滑飞行
   - 到达目标位置
   - 出现一个3D标记（上下浮动+旋转）
```

**控制台输出**：
```
🗺️ 地图点击点位，通知3D场景: 张伟
📍 计算的3D坐标: {x: -20.5, z: 15.3, type: "personnel"}
🗺️ 地图选择点位，3D场景响应: 张伟
📍 3D场景定位到坐标: {x: -20.5, z: 15.3, name: "张伟"}
```

### 2️⃣ 3D场景 → 地图联动

**功能**：点击3D场景中的建筑物，地图会自动响应

**效果**：
- 🏢 3D建筑物**高亮发光**（发光强度增加）
- 🗺️ 地图**自动飞行**到对应区域
- 📍 地图**自动放大**到建筑物位置
- 💚 地图上的建筑区域**高亮显示**（绿色边框+半透明填充）
- 📊 左上角信息面板显示建筑名称
- ⏱️ 飞行动画时长1秒

**测试步骤**：
```
1. 在3D场景中用鼠标旋转视角
2. 找到蓝色半透明的建筑物（A栋/B栋/C栋/D栋/E栋）
3. 点击任意一栋建筑
4. 观察变化：
   - 建筑物开始发光（亮度增加）
   - 左上角信息面板显示建筑名称
   - 地图开始平滑移动
   - 地图中心移动到该建筑对应区域
   - 地图上的建筑区域高亮（绿色）
```

**控制台输出**：
```
🏢 3D场景点击楼宇，通知地图: A栋
🏢 3D场景选择建筑，地图联动高亮: A栋
📍 地图定位到建筑: {lng: 113.262, lat: 23.127, name: "A栋"}
```

## 🔍 调试技巧

### 查看联动日志
```
F12 打开开发者工具 → Console标签
```

### 如果看不到点位
```
1. 检查控制台是否有"✅ 模拟数据已生成"
2. 尝试缩小地图（滚轮向下滚动）
3. 刷新页面重新加载
```

### 如果联动不工作
```
1. 检查控制台是否有错误信息
2. 确认点击的是点位或建筑（不是空白区域）
3. 观察控制台日志，确认事件被触发
```

### 如果3D场景卡顿
```
1. 关闭其他浏览器标签页
2. 使用Chrome浏览器（性能最佳）
3. 检查显卡驱动是否更新
```

## 📝 已知限制

1. **坐标映射精度**：当前使用简化的线性映射，实际项目中需要使用更精确的坐标转换
2. **标记持久化**：多次点击会累积标记，实际项目中应该限制标记数量
3. **建筑数量**：当前只有5栋建筑，实际项目中可以加载更多
4. **动画中断**：快速连续点击可能导致动画中断
