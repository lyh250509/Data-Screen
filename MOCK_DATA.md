# 模拟数据系统说明

## 概述

为了方便演示和开发，本项目内置了完整的模拟数据系统，无需连接真实后端服务即可运行完整的数据大屏。

## 数据类型

### 1. 统计数据
- **在园人员**：156人（动态更新）
- **在园车辆**：89辆（动态更新）
- **设备总数**：324台
- **在线设备**：~312台（会动态变化）
- **告警数量**：实时累计

### 2. 人员数据
- 包含156条人员记录
- 每条记录包括：ID、姓名、部门、位置、状态
- 部门包括：研发部、生产部、质检部、市场部等10个部门
- 每3秒更新5-15人的位置，模拟人员移动
- 约5%的人员会显示为"warning"状态

### 3. 车辆数据
- 包含89条车辆记录
- 车辆类型分布：
  - 小型车：~45辆（50%）
  - 中型车：~28辆（30%）
  - 大型车：~12辆（15%）
  - 特种车：~4辆（5%）
- 每2秒更新3-10辆车的位置和速度
- 车牌号格式：粤A-F + 字母 + 4位数字 + 字母/数字

### 4. 设备数据
- 包含324条设备记录
- 设备类型：
  - 监控摄像头
  - 环境传感器
  - 门禁系统
  - 消防设备
  - 智能照明
  - 空调系统
  - 能耗监测
- 设备状态：
  - 在线：~96%
  - 离线：~2%
  - 故障：~2%
- 每10秒随机改变1-3个设备的状态

### 5. 建筑物数据
- 包含8栋建筑
- 建筑类型：A栋办公楼、B栋研发中心、C栋生产车间、D栋仓储中心等
- 每栋建筑包含高度、楼层等信息

### 6. 告警数据
- 每5-15秒随机生成一条告警
- 告警级别：
  - **高级**：火警预警、入侵检测、设备故障、人员未授权进入
  - **中级**：温度异常、湿度超标、门禁异常、设备离线、能耗超限
  - **低级**：设备维护提醒、定期检查提醒、耗材更换提示、系统更新通知
- 告警会显示在右侧告警列表中，最多保留50条

### 7. 人员流量趋势
- 24小时人员流量数据
- 模拟真实场景：
  - 早高峰（7-9点）：100-150人
  - 工作时间（9-12点、14-18点）：120-150人
  - 午休（12-14点）：80-100人
  - 晚高峰（18-19点）：90-130人
  - 夜间（19-7点）：10-30人

## 技术实现

### 核心文件

1. **src/utils/mockData.ts**
   - 数据生成器核心逻辑
   - 提供各类数据的生成函数
   - 包含数据更新和统计函数

2. **src/utils/websocket.ts**
   - `createMockWebSocket` 函数模拟WebSocket数据推送
   - 定时发送各类数据更新消息
   - 支持清理和停止功能

3. **src/views/Dashboard/index.vue**
   - 接收模拟数据消息
   - 分发数据到各个组件
   - 更新统计和图表

### 数据流程

```
mockData.ts (生成数据)
    ↓
websocket.ts (模拟推送)
    ↓
Dashboard (接收消息)
    ↓
├─ dataStore (统计数据)
├─ mapStore (地图点位)
├─ DeviceChart (设备图表)
├─ PersonnelChart (人员图表)
├─ VehicleChart (车辆图表)
└─ AlertList (告警列表)
```

## 使用方法

### 运行项目

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

项目启动后，模拟数据系统会自动开始工作，你会在控制台看到：

```
✅ 模拟数据已生成：
  - 人员数据：156 条
  - 车辆数据：89 条
  - 设备数据：324 条
```

### 查看实时更新

打开浏览器开发者工具的Console，可以看到实时的数据更新日志。

### 停止模拟数据

当组件卸载或页面关闭时，模拟数据系统会自动停止：

```
🛑 模拟数据源已停止
```

编辑 `src/utils/websocket.ts` 中的数据生成调用：


编辑 `src/utils/websocket.ts` 中的定时器间隔：

```typescript

// 车辆位置更新频率（默认2秒）

// 设备状态更新频率（默认10秒）
```

## 连接真实后端

当需要连接真实后端时，只需修改 `src/views/Dashboard/index.vue`：

```typescript
// 将这行：
cleanupMockWS = createMockWebSocket((message) => {...})

// 替换为：
const wsManager = new WebSocketManager({
  url: 'ws://your-backend-url/ws',
  onMessage: (message) => {
    // 使用相同的消息处理逻辑
  }
})
wsManager.connect()
```

确保后端返回的消息格式与模拟数据一致即可。

## 故障排查

### 数据不更新
1. 检查浏览器控制台是否有错误
2. 确认是否看到"模拟数据已生成"的日志
3. 刷新页面重试

### 图表显示异常
1. 确认图表组件已正确添加 ref
2. 检查数据格式是否正确
3. 查看ECharts控制台警告

### 性能问题
如果数据量太大导致性能问题，可以：
1. 减少数据总量
2. 降低更新频率
3. 减少单次更新的数据条数
