# 联动功能修复说明

## 🔧 已修复的问题

### 1. ✅ 建筑位置精确对应
**问题**：之前3D建筑位置和地图建筑位置不对应

**修复**：
- 使用统一的坐标映射公式
- 3D坐标 → 地图坐标：`lng = center[0] + (x + 20) / 10000`
- 地图上的5栋建筑现在精确对应3D场景中的建筑位置

**对应关系**：
```
A栋 (building-1): 3D(-20, -20) → 地图中心偏左下
B栋 (building-2): 3D(0, -20)   → 地图中心偏下
C栋 (building-3): 3D(20, -20)  → 地图中心偏右下
D栋 (building-4): 3D(-20, 0)   → 地图中心偏左
E栋 (building-5): 3D(20, 0)    → 地图中心偏右
```

### 2. ✅ 取消选择时清理高亮状态
**问题**：点击多个建筑后，所有建筑都变成绿色

**修复**：
- 添加 `clearBuildingHighlight()` 函数
- 当取消选择时（点击空白处），自动清除所有高亮
- 每次高亮新建筑前，先重置所有建筑样式

**逻辑**：
```typescript
// 取消选择时
sceneStore.selectedBuildingId = null
→ 触发 watch 监听
→ 调用 clearBuildingHighlight()
→ 所有建筑恢复默认样式
```

### 3. ✅ 地图定位更精确
**问题**：点击3D建筑后，地图定位不准确

**修复**：
- 使用精确的坐标反推公式
- 从3D建筑的实际坐标 `position[0]` 和 `position[2]` 计算
- 地图自动飞到建筑的精确位置

## 📊 坐标映射公式

### 地图坐标 → 3D坐标
```typescript
const x = (lng - center[0]) * 10000 - 20
const z = (lat - center[1]) * 10000 - 20
```

### 3D坐标 → 地图坐标
```typescript
const lng = center[0] + (x + 20) / 10000
const lat = center[1] + (z + 20) / 10000
```

### 坐标系说明
- **地图中心**：`[113.264, 23.129]`（广州）
- **3D场景中心**：`(0, 0, 0)`
- **映射偏移**：`20`（确保所有坐标为正）
- **缩放因子**：`10000`（将经纬度差值放大到3D空间）
