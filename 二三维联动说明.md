# 二三维联动功能说明

## 概述

本项目实现了2D地图和3D场景的双向联动功能，提供更加沉浸式的数据可视化体验。

## 功能特性

### 1. 数据区域提示

#### 2D地图数据指示器
- **位置**：地图右上角
- **显示内容**：
  - 人员位置数量（绿色指示灯）
  - 车辆位置数量（紫色指示灯）
  - 设备位置数量（蓝色指示灯）
  - 数据覆盖区域：广州园区坐标
  - 实时更新状态提示（闪烁的绿色指示灯）

#### 3D场景信息面板
- **位置**：3D场景左上角
- **显示内容**：
  - 建筑数量统计（5栋）
  - 当前选中的建筑信息
  - 二三维联动状态（旋转图标）
  - 操作提示：点击建筑查看详情

### 2. 二三维联动机制

#### 地图 → 3D场景联动
**触发方式**：点击2D地图上的点位标记

**联动效果**：
1. 地图上选中的点位会高亮显示
2. 控制台输出：`🗺️ 地图点击点位，通知3D场景: [点位名称]`
3. 计算并显示对应的3D坐标
4. 3D场景可以定位到对应区域（预留接口）

**实现原理**：
```typescript
// 2D坐标 → 3D坐标映射
const x = (point.position[0] - mapStore.center[0]) * 10000 - 20
const z = (point.position[1] - mapStore.center[1]) * 10000 - 20
```

#### 3D场景 → 地图联动
**触发方式**：点击3D场景中的建筑物

**联动效果**：
1. 被点击的建筑物会高亮（发光强度增加）
2. 3D场景信息面板显示建筑名称
3. 控制台输出：`🏢 3D场景点击楼宇，通知地图: [建筑名称]`
4. 地图可以定位到对应区域（预留接口）

**实现原理**：
```typescript
// 使用Raycaster检测点击的3D对象
raycaster.setFromCamera(mouse, camera)
const intersects = raycaster.intersectObjects(scene.children, true)
// 高亮选中的建筑
material.emissiveIntensity = 0.8
```

### 3. 状态同步

#### Pinia Store管理
两个场景通过Pinia状态管理实现数据同步：

**mapStore（地图状态）**
- `points`：地图上的所有点位
- `selectedPointId`：当前选中的点位ID
- `center`：地图中心坐标

**sceneStore（3D场景状态）**
- `buildings`：3D场景中的所有建筑
- `selectedBuildingId`：当前选中的建筑ID
- `scene`、`camera`：Three.js对象引用

#### Watch监听
```typescript
// 地图组件监听3D场景的建筑选择
watch(() => sceneStore.selectedBuildingId, (buildingId) => {
  // 同步地图视图
})

// 3D场景组件监听地图的点位选择
watch(() => mapStore.selectedPointId, (pointId) => {
  // 同步3D场景视图
})
```

## 扩展开发

### 增强联动效果

#### 1. 地图自动定位
```typescript
// 在Map2D组件中
watch(() => sceneStore.selectedBuildingId, (buildingId) => {
  if (buildingId && map) {
    const building = sceneStore.buildings.find(b => b.id === buildingId)
    if (building) {
      // 根据建筑3D坐标反推地图坐标
      const lng = mapStore.center[0] + (building.position[0] + 20) / 10000
      const lat = mapStore.center[1] + (building.position[2] + 20) / 10000
      
      // 地图定位到该坐标
      map.getView().animate({
        center: fromLonLat([lng, lat]),
        zoom: 17,
        duration: 500
      })
    }
  }
})
```

#### 2. 3D场景相机定位
```typescript
// 在Scene3D组件中
watch(() => mapStore.selectedPointId, (pointId) => {
  if (pointId && camera) {
    const point = mapStore.points.find(p => p.id === pointId)
    if (point) {
      // 计算3D坐标
      const x = (point.position[0] - mapStore.center[0]) * 10000 - 20
      const z = (point.position[1] - mapStore.center[1]) * 10000 - 20
      
      // 相机定位动画
      const targetPosition = new THREE.Vector3(x, 30, z)
      animateCameraTo(targetPosition)
    }
  }
})

function animateCameraTo(target: THREE.Vector3) {
  // 使用Tween.js或GSAP实现平滑动画
  const duration = 1000
  const startPosition = camera.position.clone()
  const startTime = Date.now()
  
  function animate() {
    const elapsed = Date.now() - startTime
    const progress = Math.min(elapsed / duration, 1)
    
    camera.position.lerpVectors(startPosition, target, progress)
    camera.lookAt(target.x, 0, target.z)
    
    if (progress < 1) {
      requestAnimationFrame(animate)
    }
  }
  
  animate()
}
```

#### 3. 添加3D标记
```typescript
// 在3D场景中添加人员/车辆标记
function addMarker(position: {x: number, z: number}, type: string) {
  const geometry = new THREE.SphereGeometry(1, 16, 16)
  const material = new THREE.MeshBasicMaterial({
    color: type === 'personnel' ? 0x43e97b : 0x667eea
  })
  
  const marker = new THREE.Mesh(geometry, material)
  marker.position.set(position.x, 2, position.z)
  marker.userData = { type: 'marker' }
  
  scene.add(marker)
  return marker
}
```

## 性能优化

1. **防抖/节流**：地图移动和3D场景旋转事件使用节流
2. **按需渲染**：3D场景仅在交互时渲染
3. **点位聚合**：大量点位时使用聚合算法
4. **LOD技术**：3D场景根据距离调整模型细节

## 常见问题

### Q: 为什么点击地图点位后3D场景没有变化？
A: 当前版本只实现了状态同步和日志输出，相机定位动画需要额外开发（参考上面的扩展开发章节）。

### Q: 如何调整坐标映射关系？
A: 修改Map2D组件中的坐标转换公式，调整缩放系数和偏移量。

### Q: 可以添加更多类型的标记吗？
A: 可以，在mockData.ts中添加新的数据类型，在Map2D和Scene3D中添加对应的渲染逻辑。

### Q: 联动是实时的吗？
A: 是的，通过Pinia的响应式状态和Vue的watch监听实现实时联动。
